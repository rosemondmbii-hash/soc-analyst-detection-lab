# SOC Analyst Detection Lab

## 📌 Overview
This project simulates a real-world Security Operations Center (SOC) workflow, where security events are monitored, analyzed, and investigated to detect potential threats.

The lab demonstrates how a SOC analyst identifies suspicious activities, correlates events, and responds to security incidents.

---

## 🎯 Objectives
- Simulate security events such as brute force attacks and unauthorized access
- Analyze system and authentication logs
- Identify Indicators of Compromise (IOCs)
- Perform incident investigation and reporting
- Apply SOC analyst methodologies in a controlled lab environment

---

## 🛠️ Skills Demonstrated
- Log analysis and interpretation  
- Threat detection and investigation  
- Incident response and reporting  
- Security event correlation  
- Risk assessment and prioritization  
- Understanding of SOC workflows  

---

## 🧪 Lab Scenario
In this simulation, multiple suspicious activities were detected within a domain environment:

- Repeated failed login attempts followed by a successful login (brute force attack)
- Unauthorized access to sensitive files
- Deletion of a financial document (`finance.xlsx`)
- Suspicious guest account login activity

These events were analyzed to determine the nature and impact of the potential security breach.

## 🔍 Key Findings

## 🚨 Incident Response Actions
- Suspicious IP addresses were identified and blocked  
- Compromised accounts were secured  
- Password resets were enforced  
- Multi-factor authentication (MFA) was recommended  
- Continuous monitoring measures were proposed
